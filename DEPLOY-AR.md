# دليل رفع الموقع على Hostinger (هوستنجر بيزنس)

الموقع مبني بـ **Next.js**، وهوستنجر بيزنس بيشغّل تطبيقات Node.js من خلال خاصية
**Node.js Web Apps** في لوحة hPanel. اتبع الخطوات بالترتيب.

---

## 1) البيانات السرية (مهم جدًا)

قبل الرفع، جهّز القيم دي (هتحطها في إعدادات التطبيق على هوستنجر — مش في الكود):

| المتغيّر | الشرح | مثال |
|---|---|---|
| `JWT_SECRET` | مفتاح تشفير جلسة الأدمن. **لازم تغيّره** لقيمة عشوائية طويلة | نص عشوائي 40+ حرف |
| `ADMIN_EMAIL` | إيميل دخول الأدمن | admin@redahammadacademy.com |
| `ADMIN_PASSWORD` | كلمة مرور الأدمن (للإنشاء الأول فقط) | كلمة قوية |
| `NEXT_PUBLIC_PLATFORM_URL` | رابط المنصة اللي بيتحوّل ليه | https://platform.redahammadacademy.com/ |
| `DATABASE_URL` | قاعدة البيانات | `file:./prisma/dev.db` |

> ملاحظة: قاعدة البيانات الافتراضية **SQLite** (ملف على السيرفر، مش محتاج إعداد).
> لو حبيت لاحقًا تستخدم MySQL بتاع هوستنجر، شوف القسم رقم 7.

---

## 2) إنشاء تطبيق Node.js على hPanel

1. ادخل **hPanel** → **Websites** → **Add Website** → اختر **Node.js App**.
2. اختر إصدار Node **20 أو أحدث**.
3. حدّد الدومين `redahammadacademy.com`.
4. ارفع ملفات المشروع (إمّا عن طريق ربط GitHub، أو رفع المشروع كـ ZIP، أو Git).
   - **ما ترفعش** مجلد `node_modules` ولا `.next` — هيتولّدوا على السيرفر.

---

## 3) إعداد المتغيّرات والأوامر

في إعدادات تطبيق Node على hPanel:

- **Environment Variables**: ضيف كل القيم من جدول الخطوة (1).
- **Build command**: `npm install && npm run build`
- **Start command / Startup**: `npm run start`
  - (`next start` بيستخدم البورت اللي هوستنجر بيحدده تلقائيًا عبر `PORT`).

---

## 4) تجهيز قاعدة البيانات وحساب الأدمن (مرة واحدة)

بعد أول build، افتح **Terminal/SSH** بتاع التطبيق ونفّذ:

```bash
npm run db:push   # ينشئ جداول قاعدة البيانات
npm run seed      # ينشئ حساب الأدمن + كورسات تجريبية
```

> `seed` بيستخدم `ADMIN_EMAIL` و `ADMIN_PASSWORD` اللي في المتغيّرات.

---

## 5) الدخول للوحة التحكم

- لوحة التحكم على: `https://redahammadacademy.com/admin`
- سجّل دخول بإيميل وباسورد الأدمن.
- من هناك تقدر:
  - **محتوى الصفحة الرئيسية**: تعدّل النصوص، الصور، الفيديوهات، الإحصائيات، آراء الطلاب.
  - **الكورسات**: تضيف/تعدّل/تحذف كورس (عنوان، تفاصيل، صورة، سعر، رابط المنصة).

---

## 6) ملاحظات مهمة

- **رفع الصور/الفيديو** بيتخزّن في مجلد `public/uploads` على السيرفر — متأكد إن المجلد
  ده موجود وقابل للكتابة (بيتعمل تلقائيًا عند أول رفع).
- لو عملت **إعادة نشر (redeploy)**، خلّي بالك إن ملف قاعدة البيانات `prisma/dev.db`
  والصور في `public/uploads` لازم **يفضلوا على السيرفر** (ماتمسحهمش)، عشان ما تخسرش
  المحتوى. خد نسخة احتياطية منهم قبل أي تحديث كبير.
- لتغيير كلمة مرور الأدمن لاحقًا: غيّر `ADMIN_PASSWORD` في المتغيّرات وأعد تشغيل
  `npm run seed`.

---

## 7) (اختياري) استخدام MySQL بدل SQLite

لو فضّلت قاعدة بيانات MySQL بتاعة هوستنجر:

1. أنشئ قاعدة MySQL من hPanel وخُد بيانات الاتصال.
2. في `prisma/schema.prisma` غيّر:
   ```prisma
   datasource db {
     provider = "mysql"
   }
   ```
3. ثبّت محوّل MySQL: `npm install @prisma/adapter-mariadb mariadb`
   وعدّل `src/lib/db.ts` و`prisma/seed.ts` لاستخدام `PrismaMariaDb` بدل
   `PrismaBetterSqlite3`.
4. حُط `DATABASE_URL` برابط اتصال MySQL، ونفّذ `npm run db:push` و`npm run seed`.

> النظام الافتراضي (SQLite) شغّال تمام لموقع بحجم أكاديمية، فمحتاجش الخطوة دي إلا لو
> حبيت توسّع لاحقًا.

---

## 8) استبدال موقع ووردبريس الحالي وربط الدومين الرئيسي

الوضع: عندك ووردبريس على نفس هوستنجر، والدومين `redahammadacademy.com` بيفتحه،
وعايز الموقع الجديد (Next.js) يحل محله على نفس الدومين. اتبع بالترتيب عشان **ما يحصلش
أي انقطاع أو فقدان بيانات**:

### الخطوة 1: خُد نسخة احتياطية من ووردبريس (مهم جدًا)
- من hPanel → **Files → Backups** (أو Backups في القائمة)، اعمل **نسخة كاملة** (ملفات +
  قاعدة بيانات) لموقع الووردبريس. احتفظ بيها عندك. كده تقدر ترجّعه لو احتجت.

### الخطوة 2: ارفع الموقع الجديد على سب-دومين مؤقت وجرّبه
- أنشئ سب-دومين مؤقت مثل `new.redahammadacademy.com` من hPanel.
- ارفع تطبيق Node.js (زي ما في الأقسام 2–4 فوق) وخلّيه على السب-دومين ده.
- جرّب كل حاجة على `new.redahammadacademy.com`: الصفحات، الأدمن، رفع الصور...
- **مهم:** ماتلمسش `platform.redahammadacademy.com` (مودل) — هو سب-دومين منفصل ويفضل زي ما هو.

### الخطوة 3: حوّل الدومين الرئيسي للموقع الجديد
بعد ما تتأكد إن الموقع الجديد شغّال 100% على السب-دومين:
1. في hPanel → **Websites**، روح لموقع الووردبريس على `redahammadacademy.com`.
2. اختَر إنك تشيله من الدومين الرئيسي (احذف الـ website بتاع ووردبريس **بعد** ما تكون
   أخدت الباك أب، أو غيّر الدومين بتاعه لسب-دومين تاني لو حابب تحتفظ بيه).
3. عيّن **الدومين الرئيسي `redahammadacademy.com`** لتطبيق Node.js الجديد
   (أو غيّر دومين التطبيق من السب-دومين المؤقت للدومين الرئيسي).
4. حدّث `metadataBase` و`ADMIN_EMAIL` لو لزم، وأعد تشغيل التطبيق.
5. تأكد إن **SSL/HTTPS** مفعّل للدومين الرئيسي (Hostinger بيوفّره مجانًا — فعّله من SSL).

### بعد التحويل
- `redahammadacademy.com` → الموقع الجديد ✅
- `platform.redahammadacademy.com` → مودل (زي ما هو) ✅
- لو حصل أي مشكلة، ترجّع الووردبريس من الباك أب فورًا.

> نصيحة: سيب السب-دومين المؤقت شغّال لحد ما تتأكد إن كل حاجة تمام على الدومين الرئيسي،
> وبعدين تقدر تحذفه.
